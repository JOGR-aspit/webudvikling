# W2: Intermediate Web Development

## Course Overview

Welcome to W2 - Intermediate Web Development! This course builds on the HTML and CSS skills you learned in W1 by refining your abilities with advanced HTML and CSS techniques. While you'll be introduced to a small amount of JavaScript, this module still predominantly focuses on HTML and CSS. At the end of this module, you should be able to comfortably implement a professional-looking design spec, albeit without functionality.

## What You'll Learn

By the end of this course, you will be able to:

- [ ] Extensively use semantic HTML elements appropriately
- [ ] Create and style HTML forms (input types, labels, placeholders, name attribute)
- [ ] Use selection elements (dropdowns with option, radio buttons, checkboxes)
- [ ] Add submit and reset buttons to forms
- [ ] Organize form controls with fieldset and legend
- [ ] Apply basic HTML form validation (HTML side only, no backend)
- [ ] Use CSS positioning more effectively
- [ ] Use advanced CSS selectors (>, +, ~) or unique classes convention (whichever works best for you)
- [ ] Use common CSS functions
- [ ] Use CSS custom properties (optional - encouraged to use whatever approach makes most sense to you)
- [ ] Understand when it's suitable to use CSS Grid vs other layout methods
- [ ] Design and create CSS Grid layouts (position elements, use fr units, use repeat function)
- [ ] Create responsive Grid layouts using auto-fit/auto-fill with minmax()
- [ ] Combine Flexbox and CSS Grid effectively
- [ ] Create CSS animations using 2D and 3D transforms
- [ ] Chain multiple transforms together
- [ ] Use transform-origin to affect transform behavior
- [ ] Use the transition property for smooth changes
- [ ] Use @keyframes and the animation property to create animations
- [ ] (Optional - extra credit) Toggle a class on/off a specific HTML element
- [ ] (Optional - extra credit) Register when a specific element is clicked

**Note on JavaScript**: This module includes a very brief, practical introduction to JavaScript. JavaScript is not required and is only for extra credit if included in assignments. You are not expected to understand the theory behind it - interactivity makes assignments more fun, that's all.

## Tips for Success

### Learning Intermediate HTML

- **Think semantically** - Always consider which HTML element best describes your content
- **Organize forms carefully** - Use labels, fieldsets, and legends for accessibility
- **Validate early** - Use HTML validation attributes as you build forms
- **Plan your structure** - Think about the document structure before coding

### Learning Advanced CSS

- **Choose the right tool** - Flexbox vs Grid vs positioning - each has its purpose
- **Start with the layout** - Build the structure with Grid or Flexbox first
- **Add styles later** - Position everything before adding detailed styling
- **Experiment with animations** - Try small animations and build up from there
- **Use browser tools** - Inspect elements to understand layout and styles

### Implementing Design Specs

- **Break it down** - Divide the design into manageable sections
- **Start with structure** - Build the HTML skeleton first
- **Layout before details** - Get the positioning right before fine-tuning styles
- **Check responsiveness** - Test at different breakpoints throughout
- **Iterate and refine** - Don't expect perfection on the first try

### General Advice

- **Build on W1 skills** - Everything you learned in W1 is still relevant
- **Practice design implementation** - This is a key skill for web developers
- **Take your time** - Intermediate concepts take practice to master
- **Celebrate progress** - Every working design is an achievement

## Common Challenges

### Flexbox vs Grid

- **Flexbox** is for one-dimensional layouts (rows or columns)
- **Grid** is for two-dimensional layouts (rows AND columns)
- **Both** can be used together in the same page
- **Choose** based on what you're trying to accomplish

### Animations

- **Start simple** - A small transition is better than a broken complex animation
- **Transform origin** matters - It affects how elements rotate and scale
- **Performance** - Some animations are more efficient than others (transforms are good)
- **Don't overuse** - Too many animations can be distracting

### JavaScript (Optional)

- **Remember it's optional** - You don't have to use it to succeed
- **Follow the pattern** - Copy the examples for toggling classes and click handlers
- **Don't worry about understanding everything** - Practical use is enough
- **Ask for help** - If you want to learn more, we're here to help
