# ASPIT Teaching Materials Repository

A comprehensive collection of web development teaching materials designed specifically for students with autism, ADHD, and similar neurodiverse conditions.

## About This Repository

This repository contains curriculum materials, assignments, projects, and code examples for web development courses. All materials are created with accessibility and clarity as top priorities, ensuring that students can learn effectively in a structured, supportive environment.

## Course Structure

The curriculum is organized into four course levels:

| Course | Description | Focus |
|--------|-------------|-------|
| **W1** | Introduction to Web Development | HTML & CSS only |
| **W2** | Intermediate Web Development | HTML, CSS & JavaScript introduction |
| **W3.1** | Full Stack Web Development | Backend development (currently PHP) |
| **W3.2** | WordPress Development | WordPress theme and plugin development |

## Quick Start

### For Students

1. **Navigate to your course folder**: W1/, W2/, W3.1/, or W3.2/
2. **Read the course README**: Each course has an overview file explaining what you'll learn
3. **Start with curriculum**: Begin with the materials in the `curriculum/` folder
4. **Complete assignments**: Practice what you've learned with assignments in `assignments/`
5. **Build projects**: Apply your skills with larger projects in `projects/`

### For Instructors

1. **Review [CLAUDE.md](CLAUDE.md)**: Comprehensive guidelines for content creation and accessibility
2. **Explore course folders**: Each course follows a consistent structure
3. **Use shared resources**: Check `resources/` for diagrams, checklists, and reference materials
4. **Contribute new materials**: Follow the guidelines in [docs/contributing.md](docs/contributing.md)

## Key Features

### Neurodiversity-Friendly Design

All materials are created with accessibility in mind:

- **Explicit instructions** - Clear, unambiguous language
- **Step-by-step breakdowns** - Complex tasks broken into manageable steps
- **Extensive code examples** - Well-commented, runnable code
- **Visual structure** - Clear headings, lists, and formatting
- **Validation checkpoints** - Ways to verify progress
- **Troubleshooting guides** - Common issues and solutions

### Consistent Organization

Every course follows the same structure:

- `curriculum/` - Main learning materials
- `assignments/solutions/` - Practice work with solutions
- `projects/` - Larger practical applications
- `code-examples/` - Code snippets and examples

### Course Progression

The curriculum builds progressively:

1. **W1**: Master HTML and CSS fundamentals
2. **W2**: Add JavaScript for interactivity
3. **W3.1**: Learn backend development and full-stack concepts
4. **W3.2**: Apply skills to WordPress development

## Accessibility Commitment

This repository is designed with neurodiversity in mind. All materials follow the accessibility principles outlined in [docs/accessibility.md](docs/accessibility.md) and [CLAUDE.md](CLAUDE.md).

- **Clear communication** - Explicit, unambiguous instructions
- **Concrete examples** - Show before explaining

- **Visual aids** - Diagrams, checklists, and visual structure
- **Supportive feedback** - Troubleshooting and validation steps
