# W1: Introduction to Web Development

## Course Overview

Welcome to W1 - Introduction to Web Development! This course focuses on the fundamentals of building web pages using **HTML** and **CSS** only. You'll learn how to create beautiful, structured web pages from scratch, understand how the web works, and develop a foundation in basic web design principles.

## What You'll Learn

By the end of this course, you will be able to:

- [ ] Use browser DevTools (element inspector, console for errors, responsive preview)
- [ ] Understand internet basics (clients, servers, and routers)
- [ ] Create HTML documents with proper structure (DOCTYPE, lang, charset, head, title, body, html)
- [ ] Use HTML tags for text (paragraphs, headings, strong/em, comments)
- [ ] Create ordered and unordered lists and understand their use cases
- [ ] Add links and images to your pages (anchor tags, attributes, file paths, img tags, alt text, resizing)
- [ ] Use classes for styling benefits
- [ ] Use semantic HTML elements (article, header, section) - encouraged but not required
- [ ] Link CSS files externally to your HTML documents
- [ ] Write CSS rules (selectors, properties, and values)
- [ ] Use CSS selectors (element, class, ID, chaining, and descendant combinators)
- [ ] Understand the cascade and specificity (rudimentary level)
- [ ] Use CSS units of measurement (px, %, vh, vw, em, rem)
- [ ] Apply common CSS properties (color, background-color, font-family, height, width, margin, padding, border)
- [ ] Understand and use the box model (padding, margin, borders, box-sizing)
- [ ] Distinguish between block and inline elements and understand normal flow
- [ ] Use `<div>` and `<span>` appropriately in your code
- [ ] Use Flexbox for flexible layouts (flex-containers, flex-items, flex-grow, flex-shrink, flex-direction, align-items, justify-content, gap)
- [ ] Create responsive layouts with media queries and breakpoints
- [ ] Use CSS positioning (with emphasis on sticky positioning for headers)
- [ ] Choose appropriate color schemes using tools like Coolors.co
- [ ] Choose appropriate fonts using sites like Fontpair or Google Fonts
- [ ] Apply gestalt principles in your designs
- [ ] Implement basic accessibility features (such as alt attributes for images)

## Prerequisites

**None!** This course is designed for beginners with no prior web development experience.

## Course Structure

### Curriculum

The `curriculum/` folder contains all learning materials:

1. **Basic Web Dev Skills** - Using DevTools and understanding how the internet works
2. **Basic HTML Structure** - HTML document structure, tags, nesting, and indentation
3. **Text in HTML** - Paragraphs, headings, strong/em tags, and comments
4. **Lists in HTML** - Ordered and unordered lists, and their practical uses
5. **Links and Images** - Anchor tags, attributes, file paths, image tags, alt text, and resizing
6. **Classes and Semantic HTML** - Using classes for styling and semantic HTML elements
7. **CSS Fundamentals** - Inline, internal, and external CSS methods; writing CSS rules
8. **CSS Selectors** - Element, class, and ID selectors; chaining; descendant combinators
9. **CSS Units and Properties** - Measurement units (px, %, vh, vw, em, rem) and common properties
10. **The Box Model** - Understanding padding, margin, borders, and box-sizing
11. **Block vs Inline Elements** - Normal flow, and using div and span appropriately
12. **Flexbox** - Creating flexible layouts with flex-containers and flex-items
13. **Responsive Design** - Media queries, breakpoints, and natural responsiveness
14. **Positioning** - CSS position property with emphasis on sticky headers
15. **Web Design Basics** - Color schemes, fonts, gestalt principles, and accessibility

## Course Policies

### Working with Code

- **Always test your code** - Open your HTML files in a browser to see the results
- **Use browser developer tools** - Press F12 to inspect elements, view errors, and test responsive views
- **Save your work frequently** - Don't lose your progress
- **Comment your code** - Explain what each section does (for your future self!)

### Using DevTools

- **Element Inspector** - View and understand the structure of your HTML elements
- **Console** - Check for errors and debug issues
- **Responsive Mode** - Test how your page looks on different screen sizes
- **Note**: You're expected to use DevTools to view your page, not to edit it

### Getting Help

If you're stuck:

1. **Re-read the curriculum** - The answer might be there
2. **Check the code examples** - Similar examples can help you understand
3. **Try the troubleshooting section** - Common issues and their fixes
4. **Compare with solutions** - See how the problem was solved
5. **Ask for help** - Don't hesitate to reach out

## Tips for Success

### Learning HTML

- **Practice frequently** - Write HTML every day if possible
- **Use semantic tags** - Choose the right tag for the job (though `<div>` with classes is also acceptable)
- **Validate your code** - Use online validators to check for errors
- **Learn by copying** - Start by copying examples, then modify them

### Learning CSS

- **Experiment freely** - Try different values to see what happens
- **Use browser tools** - Inspect elements to understand styles
- **Start simple** - Master basics before moving to advanced topics
- **Practice layout** - Spend time on the box model and flexbox

### Web Design

- **Use color tools** - Coolors.co helps you create harmonious color schemes
- **Choose readable fonts** - Fontpair or Google Fonts can help you find good combinations
- **Apply gestalt principles** - Understand how humans perceive visual elements
- **Think about accessibility** - Use alt attributes, ensure good contrast, and design for all users

### General Advice

- **Be patient** - Web development takes time to master
- **Break problems down** - Divide complex tasks into smaller steps
- **Celebrate small wins** - Every working line of code is progress
- **Ask questions** - Curiosity leads to deeper understanding
