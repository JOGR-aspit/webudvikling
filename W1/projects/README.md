# W1 Projects

This folder contains practical projects for W1 (Introduction to Web Development). Projects synthesize multiple concepts from the curriculum and are designed for students who work at approximately 50-60% of typical beginner speed.

## Project List

### Project 1: Interactive Story (Choose Your Own Adventure)

**Folder**: `project-01-interactive-story/`

**Description**: Create a multi-page interactive story where readers make choices that determine the outcome. Build a real website with multiple interconnected pages using all the HTML concepts you've learned.

**Topics Covered**:
- HTML document structure
- Text elements (headings, paragraphs, emphasis)
- Lists (choices, inventory, character info)
- Links and navigation
- Images with alt text
- Semantic HTML elements
- File organization

**Prerequisites**:
- Articles 02-06 (Basic HTML through Semantic Elements)

**Estimated Time**: 4-6 hours

**Difficulty**: Beginner

### Project 2: Travel Guide

**Folder**: `project-02-travel-guide/`

**Description**: Create a single-page travel guide for a destination of your choice. Learn to organize information using sections, lists, and images while building a practical reference page.

**Topics Covered**:
- HTML document structure
- Semantic HTML elements (header, main, section, footer)
- Heading hierarchy
- Ordered and unordered lists
- Images with descriptive alt text
- External links
- Content organization

**Prerequisites**:
- Articles 02-06 (Basic HTML through Semantic Elements)

**Estimated Time**: 2-3 hours

**Difficulty**: Beginner

## How to Use Projects

### For Students

1. **Read the project README** - Each project has detailed instructions
2. **Plan before coding** - Think through your approach first
3. **Use the starter files** - Start with the provided templates
4. **Reference the example solution** - If you get stuck, see how it's done
5. **Test frequently** - Open your pages in a browser often
6. **Check the success criteria** - Make sure you meet all requirements

### For Instructors

1. **Review project materials** - Understand the scope and requirements
2. **Adapt as needed** - Modify to fit your specific needs
3. **Use example solutions** - As reference or teaching aids
4. **Encourage creativity** - Students can personalize themes and content

## Project Guidelines

### Time Expectations

Projects are designed for students who work at approximately 50-60% of typical beginner speed. If you find yourself finishing much faster or slower, talk to your instructor.

### Getting Help

If you get stuck:

1. **Re-read the curriculum** - The answer might be there
2. **Check the example solution** - See how similar problems are solved
3. **Review code examples** - Look at patterns from the curriculum
4. **Ask for help** - Don't hesitate to reach out to your instructor

### Success Criteria

Each project has specific success criteria. Before submitting:

- [ ] Read through all success criteria
- [ ] Test that everything works
- [ ] Check that all links work correctly
- [ ] Verify images have proper alt text
- [ ] Ensure code has helpful comments

---

**Note**: Projects in W1 focus on HTML only (no CSS or JavaScript). You'll add styling and interactivity in later courses.
